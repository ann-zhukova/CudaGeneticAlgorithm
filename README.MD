### 1. Постановка задачи

**Генетический алгоритм (ГА)** - метод поиска и оптимизации, основанный на принципах естественного отбора и генетики. 
Он используется для нахождения приближенных решений сложных задач, где пространство поиска слишком велико.

**Что является «особью» (Individual):**
В данной задаче «хромосома» особи - набор из 5 вещественных чисел (генов), которые являются коэффициентами полинома 4-й степени:

$$y = a_0 + a_1x + a_2x^2 + a_3x^3 + a_4x^4$$

**Целевая функция:**
Мы ищем такие значения коэффициентов $a_0, a_1, a_2, a_3, a_4$, при которых этот полином лучше всего аппроксимирует облако из 750 экспериментальных точек.

Точки задаются по формуле:

$$y = 1.2x^2 - 2.5x + 3.0 + \epsilon$$

Где:$1.2, -2.5, 3.0$ — это истинные коэффициенты, которые генетический алгоритм пытается найти.

$\epsilon$ — случайный шум (погрешность), имитирующий реальные измерения.

**Fitness функция:**
Алгоритм стремится **минимизировать**  значение максимальной ошибки:

$$Fitness = \epsilon = \max_{x_i \in N} |f(x_i) - g_{C_i}(x_i)|$$

где $f(x_i)$ — значение полинома, а $y_i$ — реальная координата точки.


### 2. Распараллеливание на GPU

Использование графического процессора позволяет значительно ускорить работу ГА, так как многие его этапы независимы для каждой особи. 

Были распараллелены следующие части:

1.  **Вычисление Fitness (Ядро `fitnessKernel`):**
    * Это самая ресурсоёмкая часть. На CPU ошибка вычисляется для каждой особи последовательно.На GPU создаётся столько потоков, сколько особей в популяции (popSize). Каждый поток параллельно вычисляет максимальную абсолютную ошибку для «своей» особи, проходя по всем точкам (nPts). 

2.  **Сортировка (Библиотека `Thrust`):**
    * Для отбора лучших особей (Selection) популяцию нужно отсортировать. Была использована `thrust::sort`, которая выполняет быструю сортировку на GPU, используя эффективные алгоритмы (например, Radix Sort или Merge Sort).

3.  **Генерация случайных чисел (Библиотека `CURAND`):**
    * В отличие от CPU, где используется один генератор `rand()`, на GPU каждому потоку выделяется собственное состояние `curandState`. Это позволяет генерировать тысячи случайных чисел одновременно для процессов мутации и скрещивания без взаимных блокировок.

4. **Эволюция (ядро `evolveKernel`):**
    * Процессы скрещивания (crossover) и мутации выполняются параллельно для всей «худшей» половины популяции (индексы от popSize/2 до popSize-1). Для каждой такой особи:

    - Случайно выбираются два родителя из лучшей половины (индексы от 0 до popSize/2-1).

    - Выполняется одноточечный кроссовер (single-point crossover) на уровне битов: определяется случайная точка разрыва cp от 1 до 159 (так как геном каждой особи содержит 5 чисел по 32 бита, всего 160 бит). Для каждого из 5 целочисленных представлений генов биты копируются от первого родителя до точки разрыва, а остальные — от второго.

    - С вероятностью 0.1 происходит мутация случайного бита в случайном гене.
